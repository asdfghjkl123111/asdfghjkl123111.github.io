说说你对盒子模型的理解？

2019/3/28 html

​ 当对一个文档进行布局的时候，浏览器的渲染引擎会根据标准之一的css模型将所有的元素表示为一个矩形的盒子。

一个盒子由````padding，border，margin，content```组成

border：边框，围绕元素内容的内边距的一条或者是多条线，由粗细，样式，颜色三部分组成。

padding：内边距，清除内容周围的区域，内边距是透明的，取值是不能为负，能被盒子的background影响。

margin：外边距，在元素外创建额外的空白，但是空白通常指不能放其他元素的区域。

我们在查看浏览器元素的时候就会发现，元素的大小已经变成了240.

这是因为在css中盒子模型可以分为，w3c标准盒子模型，ie怪异盒子模型。

w3c标准盒子模型

由此，盒子的宽度 = width+padding+border+margin

​ 盒子的高度 = height + padding + border+margin

也就是width/height 只是内容的高度不包含padding和border值

因此上边盒子的实际宽高就变成了240

ie怪异盒子模型


盒子的宽度 = width + margin

盒子的高度 = height+margin

也就是width/height 包含了padding和border

css中的box-sizing 属性定义了引擎如何计算一个元素的总宽度和总高度。

box-sizing : content-box | border-box | inherit ;

content-box是默认值，元素的width/height 不包含padding、border，与标准盒子表现一致。

border-box元素的width/height包含padding、border，和怪异盒子保持一致。

inherit指定了box-sizing属性的值，应该从父元素继承。

这个时候，在回到浏览器查看盒子的宽度就是200px
